Connecting to Real GCP BigQuery

By default, if the application doesn't detect active Google Cloud Platform (GCP) credentials, it runs in Demo Mode using mock data.

1

Set up a Google Cloud Project

Enable the BigQuery API in the GCP Console. You need a project with active billing or the free tier active to execute queries.

2

Configure Authentication

Choose one of the following methods to authorize the app client:

Application Default Credentials (Recommended)
Service Account Key JSON

Install the Google Cloud CLI and run the following command in your terminal:

gcloud auth application-default login

This is the safest method for local development, making credentials automatically available to Flask.

Create a Service Account with the BigQuery User role, download its key file (JSON), and set the environment variable pointing to its path:

# PowerShell (Windows)
$env:GOOGLE_APPLICATION_CREDENTIALS="C:\path\to\your-key.json"

# CMD (Windows)
set GOOGLE_APPLICATION_CREDENTIALS="C:\path\to\your-key.json"

# Bash (Linux/macOS)
export GOOGLE_APPLICATION_CREDENTIALS="/path/to/your-key.json"
3

Restart the Server

Restart the Flask server. It will automatically detect your credentials and pull live, up-to-date release feeds directly from Google Cloud BigQuery.
